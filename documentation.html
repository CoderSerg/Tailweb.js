Tailweb.js

A lightweight UI framework you can drop into any website via the DevTools console. Loads Tailwind CSS automatically — no setup, no build step.

What is it?

Tailweb lets you build fully styled floating windows with buttons, labels, inputs, toggles, popups, and notifications — all from the browser console. Drop tailweb.js in, and start building immediately.

Features

  • Draggable, resizable windows with accent color theming
  • Buttons, labels, text inputs, toggles, and dividers
  • Key system — lock windows behind a key screen
  • Global popups with up to 3 buttons
  • Toast notifications with a progress bar
  • Lucide icon support on buttons, labels, and popups
  • Chainable API

Quick Start

Get Tailweb running in seconds.

Load it

Paste tailweb.js into the DevTools console. You can't inject it via a bookmarklet due to Content Security Policy on most sites.

Once loaded, tailweb is available on window.

First window

const win = tailweb.Window({
  title: "My Tool",
  tailwindAccentColor: "cyan-500"
})

win.Label({ title: "Hello, Tailweb!", size: 14 })
win.Button({ title: "Click me", callback: () => alert("yo") })

Window

Creates a draggable, resizable floating window. All elements are added to it.

Syntax

const win = tailweb.Window({
  title,
  tailwindAccentColor,
  key,
  getKey
})

Properties

PropertyTypeDefaultDescription
titlestring"Tailweb"Title shown in the window header
tailwindAccentColor optionalstring"gray-500"Accent color — Tailwind name or hex string
key optionalstring | array—Lock the window behind a key screen on load. Pass a string or ["key1","key2"] for multiple valid keys
getKey optionalfunction—Called when "Get Key" is clicked on the key screen

Accent Colors

Pass a Tailwind color name or any hex string like "#ff6b6b".

gray-500blue-500red-500green-500purple-500yellow-500pink-500orange-500cyan-500indigo-500teal-500violet-500

Example

const win = tailweb.Window({
  title: "My Hub",
  tailwindAccentColor: "violet-500",
  key: ["secret", "devkey"],
  getKey: () => window.open("https://yoursite.com/getkey")
})

Key System

Lock your window behind a key screen on load.

How it works

When key is passed to tailweb.Window(), a key screen overlays the window immediately. The user must enter a valid key to access the window.

You can pass a single key as a string, or multiple valid keys as an array — any of them will unlock the window.

Key Screen Buttons

ButtonDescription
ExitRemoves the window entirely
Get Key optionalOnly shown if getKey is provided. Calls the function you passed
SubmitValidates the key — dismisses screen if correct, flashes red if wrong

All key screen elements follow the window's accent color.

Example

// Single key
const win = tailweb.Window({
  title: "Locked",
  key: "supersecret",
  getKey: () => window.open("https://yoursite.com/key")
})

// Multiple keys
const win = tailweb.Window({
  title: "Locked",
  key: ["key1", "key2", "devmode"]
})

Re-locking

Use win.Lock() to re-show the key screen at any time. See the Lock page.

Properties

TextBox

Adds a text input. Binds to a ref object and updates live or on blur.

Syntax

win.TextBox({
  title,
  placeholder,
  variable,
  value,
  updateBlur
})

Properties

PropertyTypeDefaultDescription
titlestring—Label shown above the input
placeholder optionalstring""Placeholder text inside the input
variable optionalref object—Ref object to bind the value to
value optionalstring""Default pre-filled value
updateBlur optionalbooleanfalsefalse = update on every keystroke, true = update only on blur

Variable Binding

JS primitives can't be passed by reference, so use a ref object:

const myVar = { __tailwebRef: { value: "" } }

win.TextBox({ title: "Name", variable: myVar })

console.log(myVar.__tailwebRef.value) // read current value

Example

const query = { __tailwebRef: { value: "" } }

win.TextBox({
  title: "Search",
  placeholder: "Type to search...",
  variable: query,
  value: "default text",
  updateBlur: true
})

Toggle

Adds an on/off switch. Fires a callback with true or false every time it's flipped.

Syntax

win.Toggle({
  title,
  value,
  callback
})

Properties

PropertyTypeDefaultDescription
titlestring—Label shown next to the toggle
value optionalbooleanfalseInitial state
callback optionalfunction—Called with true/false when flipped

Example

win.Toggle({
  title: "Enable feature",
  value: false,
  callback: (state) => console.log("on:", state)
})

// with a named function — no parentheses
function onToggle(state) { console.log(state) }
win.Toggle({ title: "Toggle", callback: onToggle })

Lock

Re-shows the key screen, locking the window again.

Syntax

win.Lock()

Notes

Does nothing if no key was set on the window, or if the window is already locked.

Example

win.Button({
  title: "Lock",
  icon: "lock",
  callback: () => win.Lock()
})

Remove

Destroys the window and all its elements.

Syntax

win.Remove()

Notes

tailweb stays available after Remove() — you can create a new window anytime.

Example

win.Button({
  title: "Close",
  icon: "x",
  iconColor: "red-500",
  callback: () => win.Remove()
})

Popup

Shows a centered modal with a dark overlay. Accent color is pulled from a random existing window.

Syntax

tailweb.Popup({
  title,
  description,
  icon,
  button1,
  button2,
  button3,
  cbutton1,
  cbutton2,
  cbutton3
})

Properties

PropertyTypeDefaultDescription
titlestring—Popup title
description optionalstring—Body text below the title
icon optionalstring—Lucide icon shown on button1
button1string—Primary button (accent-tinted)
button2 optionalstring"Cancel"If omitted, auto-adds a Cancel button that closes the popup
button3 optionalstring—Third button
cbutton1 optionalfunction—Called when button1 is clicked
cbutton2 optionalfunction—Called when button2 is clicked
cbutton3 optionalfunction—Called when button3 is clicked

Buttons render right-to-left: button3, button2/Cancel, button1.

Example

// Simple confirm
tailweb.Popup({
  title: "Are you sure?",
  description: "This can't be undone.",
  button1: "Continue",
  icon: "arrow-right",
  cbutton1: () => console.log("confirmed")
})

// Three buttons
tailweb.Popup({
  title: "Save changes?",
  button1: "Save",
  button2: "Discard",
  button3: "Cancel",
  cbutton1: () => console.log("saved"),
  cbutton2: () => console.log("discarded")
})

Notify

Shows a toast notification in the bottom right. The accent-colored progress bar shrinks right to left over the duration.

Syntax

tailweb.Notify({
  title,
  description,
  icon,
  time
})

Properties

PropertyTypeDefaultDescription
titlestring—Notification title
description optionalstring—Body text below the title
icon optionalstring—Lucide icon name, shown at 32px
time optionalnumber3Seconds before auto-dismiss

Example

tailweb.Notify({
  title: "Saved",
  description: "Your changes were saved.",
  icon: "check-circle",
  time: 4
})

Icons

Lucide icons on buttons, labels, popups, and notifications — fetched automatically.

How it works

Pass any valid Lucide icon name to the icon property. Tailweb fetches the SVG from the CDN and injects it inline — no setup needed.

  • Icons render to the left of text
  • Size matches the element's font size
  • Default color is white unless iconColor is set
  • iconColor accepts the same Tailwind names or hex strings as tailwindAccentColor
  • Icons are fetched async — they pop in a moment after the element renders

Supported on

  • win.Button() — icon and iconColor
  • win.Label() — icon and iconColor
  • tailweb.Popup() — icon on button1 only
  • tailweb.Notify() — icon at 32px

Example

win.Button({ title: "Delete", icon: "trash-2", iconColor: "red-500" })
win.Button({ title: "Save", icon: "save" })
win.Label({ title: "Settings", icon: "settings", size: 13 })
win.Label({ title: "Warning", icon: "triangle-alert", iconColor: "yellow-500" })

Chaining

All element methods return the window instance so you can chain calls.

Example

tailweb.Window({ title: "Toolbox", tailwindAccentColor: "violet-500" })
  .Label({ title: "Config", icon: "settings" })
  .TextBox({ title: "API Key", placeholder: "sk-..." })
  .Divider()
  .Toggle({ title: "Enable", callback: (s) => console.log(s) })
  .Button({ title: "Submit", icon: "check", callback: () => {} })
  .Key({ key: "F1" })

Callbacks

How to correctly pass functions as callbacks.

Always pass, never call

Pass the function reference — no parentheses. Adding () calls the function immediately at the time the element is created, and passes its return value instead of the function itself.

// ✅ correct — passes the function
win.Button({ title: "Hi", callback: myFunc })
win.Toggle({ title: "Toggle", callback: myFunc })

// ✅ also correct — arrow function
win.Button({ title: "Hi", callback: () => myFunc() })

// ❌ wrong — calls immediately, passes return value
win.Button({ title: "Hi", callback: myFunc() })

Toggle callbacks

Toggle passes the current state as a boolean argument to your callback:

function onFlip(state) {
  console.log("Toggle is now:", state) // true or false
}

win.Toggle({ title: "Feature", callback: onFlip })

Full Example

Everything Tailweb has, in one script.

const nameRef = { __tailwebRef: { value: "" } }

const win = tailweb.Window({
  title: "My Hub",
  tailwindAccentColor: "cyan-500",
  key: ["letmein", "devmode"],
  getKey: () => window.open("https://yoursite.com/getkey")
})

win.Label({ title: "Enter your info", icon: "clipboard", size: 11 })
win.TextBox({ title: "Name", placeholder: "John Doe", variable: nameRef })
win.Divider()
win.Toggle({
  title: "Notify on submit",
  value: true,
  callback: (s) => console.log("notify:", s)
})
win.Button({
  title: "Submit",
  icon: "check",
  callback: () => {
    tailweb.Popup({
      title: "Submitted!",
      description: "Name: " + nameRef.__tailwebRef.value,
      button1: "OK",
      cbutton1: () => {
        win.Remove()
        tailweb.Notify({ title: "Done", icon: "check-circle", time: 2 })
      }
    })
  }
})
win.Button({ title: "Lock", icon: "lock", callback: () => win.Lock() })
win.Button({ title: "Close", icon: "x", iconColor: "red-500", callback: () => win.Remove() })
win.Key({ key: "F2" })
