Tailweb.js

A lightweight UI framework for any website. Drop it in the console, build UIs instantly.

What is Tailweb?

Tailweb is a browser-side UI framework you can inject into any website via DevTools. It loads Tailwind CSS automatically — no build step, no bundler, no setup.

Once loaded, you get a full set of draggable windows, buttons, labels, text inputs, key systems, and icon support — all styled consistently with your chosen accent color.

Features

  • Draggable, resizable windows
  • Buttons, labels, and text inputs
  • Key system with lock/unlock
  • Lucide icon support on buttons and labels
  • Accent color theming
  • Keyboard toggle keybinds
  • Chainable API

Quick Start

Get Tailweb running in seconds.

Loading Tailweb

Paste tailweb.js directly into the DevTools console. That's it — you can't eval or inject it via a bookmarklet because of Content Security Policy.

Once loaded, tailweb is available on window.

Your first window

const win = tailweb.Window({
  title: "My Tool",
  tailwindAccentColor: "cyan-500"
})

win.Label({ title: "Hello, Tailweb!", size: 14 })
win.Button({ title: "Click me", callback: () => alert("yo") })

Window

Creates a draggable, resizable window. All elements are added to it.

Syntax

const win = tailweb.Window({
  title,
  tailwindAccentColor,
  key,
  getKey
})

Properties

PropertyTypeDefaultDescription
titlestring"Tailweb"Title shown in the window header
tailwindAccentColor optionalstring"gray-500"Accent color — Tailwind name or hex string
key optionalstring—If set, shows a key screen on load
getKey optionalfunction—Called when "Get Key" is clicked on the key screen

Accent Colors

Pass a Tailwind color name or any hex string like "#ff6b6b".

gray-500blue-500red-500green-500purple-500yellow-500pink-500orange-500cyan-500indigo-500teal-500violet-500

Example

const win = tailweb.Window({
  title: "My Hub",
  tailwindAccentColor: "violet-500",
  key: "mysecretkey123",
  getKey: () => window.open("https://yoursite.com/getkey")
})

Key System

Lock your window behind a key screen on load.

How it works

When key is passed to tailweb.Window(), a key screen overlays the window on load. The user must enter the correct key to access the window.

Key screen buttons

ButtonDescription
ExitRemoves the window entirely
Get Key optionalOnly shown if getKey is provided. Calls the function you passed
SubmitValidates the key — dismisses screen if correct, flashes red if wrong

All key screen elements follow the window's accent color.

Example

const win = tailweb.Window({
  title: "My Hub",
  key: "supersecret",
  getKey: () => window.open("https://yoursite.com/key"),
  tailwindAccentColor: "cyan-500"
})

Re-locking

Use win.Lock() to re-show the key screen at any time. See the Lock page for details.

Button

Adds a clickable button to the window.

Syntax

win.Button({
  title,
  callback,
  icon,
  iconColor
})

Properties

PropertyTypeDefaultDescription
titlestring"Button"Text displayed on the button
callback optionalfunction—Function called when button is clicked
icon optionalstring—Any valid Lucide icon name e.g. "trash-2"
iconColor optionalstring"#ffffff"Icon color — Tailwind name or hex string

Example

win.Button({
  title: "Say Hello",
  callback: () => console.log("hello")
})

win.Button({
  title: "Delete",
  icon: "trash-2",
  iconColor: "red-500",
  callback: () => console.log("deleted")
})

Syntax

win.Label({
  title,
  size,
  icon,
  iconColor
})

Properties

PropertyTypeDefaultDescription
titlestring""Text content of the label
size optionalnumber11Font size in pixels
icon optionalstring—Any valid Lucide icon name e.g. "settings"
iconColor optionalstring"#ffffff"Icon color — Tailwind name or hex string

Example

TextBox

Adds a text input. Binds to a ref object and updates live or on blur.

Syntax

win.TextBox({
  title,
  placeholder,
  variable,
  value,
  updateBlur
})

Properties

PropertyTypeDefaultDescription
titlestring—Small label shown above the input
placeholder optionalstring""Placeholder text inside the input
variable optionalref object—Ref object to bind the value to
value optionalstring""Default value pre-filled in the input
updateBlur optionalbooleanfalsefalse = live updates, true = updates on blur

Variable binding

JS primitives can't be passed by reference, so use a ref object:

const myVar = { __tailwebRef: { value: "" } }

win.TextBox({
  title: "Username",
  variable: myVar,
  updateBlur: false
})

// Access the current value:
console.log(myVar.__tailwebRef.value)

Example

const query = { __tailwebRef: { value: "" } }

win.TextBox({
  title: "Search",
  placeholder: "Type to search...",
  variable: query,
  value: "default text",
  updateBlur: true
})

Key

Binds a keyboard key to toggle the window's visibility.

Syntax

win.Key({ key })

Properties

PropertyTypeDescription
keystringAny valid KeyboardEvent.key value

Example

win.Key({ key: "F2" })
win.Key({ key: "`" })
win.Key({ key: "Escape" })

Lock

Re-shows the key screen, locking the window.

Syntax

win.Lock()

Notes

Does nothing if no key was set on the window, or if the window is already locked.

Example

win.Button({
  title: "Lock",
  icon: "lock",
  callback: () => win.Lock()
})

Remove

Destroys the window and all its elements.

Syntax

win.Remove()

Notes

tailweb itself stays available after Remove() — you can create a new window at any time.

Example

win.Button({
  title: "Close Window",
  icon: "x",
  iconColor: "red-500",
  callback: () => win.Remove()
})

Icons

Lucide icons on buttons and labels, fetched automatically.

How it works

Button and Label support an icon property. Pass any valid Lucide icon name and it's fetched from the CDN and injected as an inline SVG.

  • Icons render to the left of the text
  • Size matches the element's font size
  • Default color is white unless iconColor is set
  • iconColor accepts the same Tailwind names or hex strings as tailwindAccentColor

Example

win.Button({ title: "Delete", icon: "trash-2", iconColor: "red-500", callback: () => {} })
win.Button({ title: "Save", icon: "save", callback: () => {} })
win.Label({ title: "Settings", icon: "settings", size: 13 })
win.Label({ title: "Warning", icon: "triangle-alert", iconColor: "yellow-500" })

Note

Icons are fetched async so they pop in a moment after the element renders. This is expected — it's one network request per icon.

Chaining

All element methods return the window instance so you can chain calls.

Example

const win = tailweb.Window({ title: "Toolbox", tailwindAccentColor: "violet-500" })

win
  .Label({ title: "Config", icon: "settings", size: 12 })
  .TextBox({ title: "API Key", placeholder: "sk-...", updateBlur: true })
  .Button({ title: "Submit", icon: "check", callback: () => console.log("submitted") })
  .Key({ key: "F1" })

Full Example

Everything Tailweb has, in one script.

const nameRef = { __tailwebRef: { value: "" } }
const emailRef = { __tailwebRef: { value: "" } }

const win = tailweb.Window({
  title: "My Hub",
  tailwindAccentColor: "cyan-500",
  key: "letmein",
  getKey: () => window.open("https://yoursite.com/getkey")
})

win.Label({ title: "Fill out the form below", icon: "clipboard", size: 11 })

win.TextBox({
  title: "Name",
  placeholder: "John Doe",
  variable: nameRef,
  updateBlur: false
})

win.TextBox({
  title: "Email",
  placeholder: "john@example.com",
  variable: emailRef,
  updateBlur: true
})

win.Button({
  title: "Submit",
  icon: "check",
  callback: () => {
    console.log("Name:", nameRef.__tailwebRef.value)
    console.log("Email:", emailRef.__tailwebRef.value)
  }
})

win.Button({
  title: "Lock",
  icon: "lock",
  callback: () => win.Lock()
})

win.Button({
  title: "Close",
  icon: "x",
  iconColor: "red-500",
  callback: () => win.Remove()
})

win.Key({ key: "F2" })
